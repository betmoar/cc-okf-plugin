# OKF Examples

A complete, minimal OKF bundle and the artifacts the lifecycle commands produce.

## A minimal bundle

```
my-knowledge/
├── .okf/
│   └── active
├── index.md
├── log.md
└── concepts/
    ├── spaced-repetition.md
    └── forgetting-curve.md
```

## Concept: `concepts/spaced-repetition.md`

```markdown
---
id: spaced-repetition
title: Spaced Repetition
type: concept
status: stable
created: 2026-06-01
updated: 2026-06-19
tags: [learning, memory]
links: [forgetting-curve]
sources:
  - title: "Ebbinghaus, Memory: A Contribution to Experimental Psychology"
    url: https://example.com/ebbinghaus
---

Spaced repetition schedules reviews at increasing intervals to exploit the
spacing effect, counteracting the [[forgetting-curve]]. Reviews are timed to
occur just before predicted recall failure.

## Why it works
Each successful retrieval strengthens the memory trace and flattens the curve.
```

The `links: [forgetting-curve]` frontmatter entry is **generated** by
`/okf:reindex` from the body's `[[forgetting-curve]]` wiki-link — do not
hand-edit it. Body `[[id]]` references are the source of truth; `/okf:validate`
checks them directly (a dangling body `[[id]]` is an ERROR).

## Concept: `concepts/forgetting-curve.md`

```markdown
---
id: forgetting-curve
title: Forgetting Curve
type: concept
status: stable
created: 2026-06-01
updated: 2026-06-02
tags: [learning, memory]
---

The forgetting curve describes the exponential decay of retention over time
without reinforcement. Spaced review (see [[spaced-repetition]]) slows the decay.
```

This concept does not declare `links` in frontmatter even though its body
references `[[spaced-repetition]]`. After running `/okf:reindex`, the generated
`links: [spaced-repetition]` entry will be spliced in automatically. Until then,
`/okf:validate` reports a WARN (stale `links:`) but not an ERROR — the body
wiki-link itself resolves, so referential integrity is intact.

## Typical workflow

1. `/okf:activate` — writes `.okf/active`; restart the session so the gate fires.
2. Create or edit `concepts/<id>.md` files (this is where the `okf` skill's
   conventions apply).
3. `/okf:reindex` — refresh `index.md`.
4. `/okf:log "what changed"` — record the change.
5. `/okf:validate` — confirm conformance (expect `0 error(s)`).

## A non-conformant example and its fixes

```markdown
---
id: wrong-id          # ERROR C3: filename is gamma.md, so id must be "gamma"
title: Gamma
type: bogus           # ERROR C5: not an allowed type
created: 2026-06-20
updated: 2026-06-05   # ERROR C8: updated is before created
links: [does-not-exist]   # WARN X2: links: out of sync with body — run /okf:reindex
---

Body cites [[also-missing]].   # ERROR X1: body wiki-link does not resolve
```

Fixes: rename the file to match `id` (or fix `id`), set `type` to an allowed
value, correct the dates so `updated >= created`, and repoint or remove the
dangling `links`/`[[wiki-link]]` targets. Then `/okf:reindex` and `/okf:validate`.
